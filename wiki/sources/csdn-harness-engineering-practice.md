## Key points

- Harness Engineering is about making agents readable, controllable, verifiable, and recoverable.
- Prompt, context, and harness are three different layers of system design.
- `AGENTS.md` is a practical mechanism for making project rules visible to agents.
- Hard constraints outperform prompt-only requests for critical behaviors.
- Validation, review separation, and lessons learned are core feedback loops.
