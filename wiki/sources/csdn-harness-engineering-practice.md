## 关键要点

- Harness Engineering 关注让 Agent 可读、可控、可验证、可恢复。
- Prompt、Context、Harness 是三层不同的问题。
- `AGENTS.md` 是让项目规则对 Agent 可读的一种实用机制。
- 对关键行为来说，硬约束比 prompt 提醒更可靠。
- 验证、执行与评审拆分、经验沉淀构成了反馈回路。
